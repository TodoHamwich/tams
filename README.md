# Todo's Advanced Modular System (TAMS)

A custom system for Foundry VTT.

## Features
- **Capped 1d100 Rolls:** 1d100maxStat + Familiarity.
- **Limb-Based Health:** Tracks 7 body regions with unique health multipliers and armor.
- **Dynamic Resource Pools:** Add custom resources that scale with stats.
- **Weapon Tags:** Automated damage calculation for Heavy, Two-Handed, and Light weapons.
- **Visual Themes:** Default, Dark, and Parchment styles.

## Installation
To install the system, paste the following manifest URL into the "Install System" dialog in the Foundry VTT Setup menu:

`https://github.com/TodoHamwich/tams/releases/latest/download/system.json`

## Development

### Tooling
- Build: Vite (`npm run build`) - bundles JS and compiles SCSS.
- CSS/SCSS: Modular partials in `styles/src/` compiled to `styles/tams.css` via `npm run sass`.
- Tests: Vitest (`npm test`)
- CI: GitHub Actions (`.github/workflows/ci.yml`) runs tests and builds on push.
- Local link to Foundry Data: `./link-to-foundry.sh` (creates symlink to `~/.local/share/FoundryVTT/Data/systems/tams`)

### Source structure (modular)
- `src/models/`: DataModels (character, items)
- `src/documents/`: Document classes (`TAMSActor`, `TAMSItem`)
- `src/applications/`: ApplicationV2 sheets (Actor/Item/Loot)
- `src/utils/`: Utilities
  - `helpers.js`: sheet/actions utilities and chat helpers
  - `combat.js`: combat UI helpers (`getHitLocation`, `showCombinedInjuryDialog`)
- `src/tams.js`: System entry (hooks, settings, registrations)
- `templates/`: Handlebars templates for Actor and Item sheets.
- `styles/`: 
  - `tams.scss`: Entry point for SCSS.
  - `src/`: Modular SCSS partials (`_variables`, `_base`, `_actor-sheet`, etc.).
  - `tams.css`: Compiled output for Foundry (generated by `npm run sass`).
- `scripts/tams.js`: Bundled output for production (generated by Vite).

### Combat and damage flow
- Chat damage application now calls `TAMSActor.applyDamage(hits, { isAoE, multiplier })` to centralize:
  - armor and “Alternate Armour” handling (including item vs. natural armor)
  - squad/horde capping and size reduction
  - pending injury/crit/unconscious/survival checks
- UI dialogs and combat messages are localized via `game.i18n`.

### Localization (i18n)
- All player-facing strings live in `lang/en.json` under `TAMS.*`.
- Templates use `{{localize 'TAMS.Key'}}`; JS uses `game.i18n.localize/format`.
- Notification strings are under `TAMS.Checks.Notifications.*`.

### Running locally
1) Install Node (v18+ recommended). On this machine, Homebrew-installed Node is used.
2) Install deps: `npm install`
3) Build: `npm run build`
4) Run tests: `npm test`
5) Start Foundry VTT, open a world, and enable the TAMS system (symlink must exist).
